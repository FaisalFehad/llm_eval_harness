- ABOVE_200K — midpoint >= £200,000

Source priority:
1. First, scan the description for a salary paragraph (look for "Compensation:" "Salary:" "£" "Package:").
2. If description has no salary but the title contains a specific £ amount or range (e.g., "£80k+", "£60-130k"), use the title salary as fallback.
3. A bare company name or marketing text in the title (e.g., "Quant Hedge Fund", "VC backed startup") does NOT count as salary — ignore.

Classification steps:
1. Non-£ currency ($, €, USD, EUR): NO_GBP. Stop.
2. Disqualifiers (always NO_GBP): OTE, commission, TC/total package, daily rate, hourly rate.
3. Per-period rates: annualize before classification.
   - Monthly: multiply by 12. Example: "£700/month" → £700 × 12 = £8,400 → BELOW_30K.
   - Weekly: multiply by 52. Example: "£1,200/week" → £1,200 × 52 = £62,400 → 60_70K.
4. Soft disqualifier without £ amount — "competitive salary", "negotiable", "DOE", "D.O.E.": NO_GBP. Stop.
   Exception: if £ amount follows (e.g., "£40-60k DOE"), use the £ amount.
5. "Up to £X" / "to £X" with no lower bound: midpoint = £X/2. Example: "Up to £90k" → midpoint £45k = RANGE_45_54K. Not UP_TO_ONLY.
   Exception: "competitive salary up to £90k" (no explicit £) = NO_GBP.
6. Single £ value or "£X and above" / "£X+": that value IS the midpoint. Pick bucket.
7. £ range: midpoint = (min + max) / 2. Use MIDPOINT, not upper bound.

Special: In UK job listings, a plain number with "k" suffix (e.g., "30K", "up to 60k") without $/€ prefix is implicitly GBP. Treat same as "£30k".
