===== STEP 4: COMPENSATION (max 25) =====

STEP 4A: Scan the ENTIRE jd_text character by character for the £ symbol.
  The £ may appear inside markdown formatting (e.g. **Salary**\n- Salary: £100,000).
  The £ may appear in bullet points, emoji-prefixed lines, or plain text.
  If NO £ symbol exists anywhere in jd_text → comp = 0 and STOP.
  If £ is found → EXTRACT the exact substring containing the salary. Write it down. Then continue to STEP 4B.

STEP 4B: Determine if the salary is a RANGE or an "up to" cap.

  FIRST, check if TWO £ amounts are present separated by "-", "–", or "to". Examples:
    "£75,000 - £85,000" → RANGE (two £ figures)
    "£90,000-£110,000" → RANGE
    "£50-100k" → RANGE (two numbers, one £)

    "£75,000 - £85,000" inside markdown → still a RANGE
  If a RANGE is found → go to STEP 4C.

  SECOND, only if ONE £ amount is present with no second number, check for "up to" or "to" preceding it.
    If "up to £X" or "Up to £X" or "to £X" with no lower bound → comp = 0 and STOP.
    If a single figure with no "up to" → treat as both min and max. Go to STEP 4C.

STEP 4C: Calculate the midpoint.
  Count ONLY base salary in GBP (£). Ignore bonuses, equity, benefits, USD ($), EUR.
  Converting "k" notation: £100k = £100,000. £80k-£120k = £80,000-£120,000.
  IMPORTANT: When "k" appears only after the second number, it applies to BOTH.
    £50-100k = £50,000-£100,000. £75-85k = £75,000-£85,000.
  If a range is given, calculate: midpoint = (lower + upper) / 2.

  +25 if midpoint >= £100,000
  +15 if midpoint £75,000-£99,999
  +5  if midpoint £55,000-£74,999
  -30 if midpoint < £45,000
